# policies01 — Estratégia de Saída (Exit Strategy)

## 1. Objetivo

Garantir portabilidade de sistemas e dados entre provedores de nuvem, fator relevante principalmente para contratos governamentais.

---

## 2. Diretrizes Técnicas

- Uso de containers (Docker)
- Orquestração Kubernetes
- Banco de dados compatível multi-cloud
- Infraestrutura como código (Terraform)

---

## 3. Política de Dados

- Backup independente do provedor
- Criptografia com chave própria
- Replicação periódica entre ambientes

---

## 4. Procedimento de Migração

1. Exportação de dados
2. Validação de integridade
3. Implantação paralela
4. Teste de carga
5. Corte controlado

---

## 5. Benefício Estratégico

Redução de dependência contratual e aumento do poder de negociação.
